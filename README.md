# Apple LP 模写

## 概要
SCSSとjQueryの練習として製作。<br>
ブレークポイントが多く、制作が複雑で、抽象的な表現にはなるが、おしゃれなWebサイトを制作するための練習。<br>
画像・フォントファミリーは [『Apple（日本）』](https://apple.com/jp/)から引用。<br>
※srcディレクトリにビルド前のコード、distディレクトリにビルド後のコードが入っています。

## 開発環境・使用言語
### 開発環境
- macOS
- VS Code
- webpack
### 使用言語
- HTML
- SCSS
- JavaScript

## 今回の学び
1. Apple公式サイトを数分眺め、無計画にコードを書き始めたため、終始コードが汚く、自分で書いているのにも関わらず、どこに何が書かれているのかを探すのに苦労した。<br>
基本的なことだが、ワイヤーを組むなど準備の大切さに改めて気付かされた。
2. 無計画にコードを書くことで普段のコーディング以上に修正力が身についた。

## 今後の改善点
1. 初めてSCSSとjQueryを書き、練習のために mixin, if, for などの制御構造を多用したことで、コードの可読性が悪い。可読性・メンテナンス性が高いコードが書けるようになる必要がある。
2. 2日で製作する予定がヘッダー部分のレスポンシブ化に手間がかかり、予定より1日多い3日を要した。製作難易度に対して、自分のスキルの見積もりが甘い。
3. polyfillを導入後、Chrome, Firefox, IE, Edgeでは正常に機能したJavaScriptのAPI Element.closest()が iOS14.4, Safari14.0.3 では機能しない。
